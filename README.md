Воспроизведение результатов статьи [Stop Regressing: Training Value Functions via Classification for Scalable Deep RL](https://arxiv.org/pdf/2403.03950).

# Описание

Авторы статьи предлагают классификационный подход к оценке Q-функции в Value-based подходах, утверждая, что регрессия хуже скейлится, чем классификация, что подтверждают примерами вроде LLM и визуальных моделей.

Ключевая идея - заменить числовую оценку Q-функции в моделях вроде DQN на дискретное вероятностное распределение, в котором значения это равномерно выбранные точки на отрезке $[v_{min}, v_{max}]$. Во время инференса можно брать матожидание или моду такого распределения.

Чтобы обучить такую модель, необходимо адаптировать числовой таргет из TD на другое распределение, здесь авторы предлагают разные подходы, мы сосредоточились на двух:

- 1. HL Gauss - сглаживание таргета через нормальное распределение
- 1. Two hot encoding - выбор двух ближайших точек к таргету на разделенном отрезке $[v_{min}, v_{max}]$ и выбор вероятности для каждой из них так, чтобы матожидание совпадало с таргетом

В качестве среды взяли классическую игру из ATARI с мячиком и платформой. 

В качестве архитектуры нейронки взяли DQN из оригинальной статьи с 3 слоями сверток и 2 линейными слоями для предсказания Q-функции.

# Код и результаты

Чтобы собрать окружение, достаточно лишь выполнить `uv sync`. В `.ipynb` ноутбуках в корневой директории лежат конфиги для запуска экспериментов и необходимый идентичный код. В `runs/` находятся копии ноутбуков с разными сидами и результаты их выполнения, а `results/` содержит статистики обучения. `report.ipynb` объединняет результаты по всем экспериментам с усреднением по сидам.

Итого, результаты воспроизводятся следующим образом: сборка окружения через `uv sync`, запуск всех ноутбуков в директории `runs/`, а далее исполнение `report.ipynb`.
